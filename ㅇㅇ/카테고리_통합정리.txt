[1] 통합 기준
- 파일 제목/헤더의 CAT 번호가 동일하거나, 동일 이벤트 코드를 반복 포함하면 중복으로 판단
- 단일 카테고리 원본 파일(예: CAT-02 전용)은 "기준 원본"으로 유지
- 다중 카테고리 혼합 파일은 "통합 인덱스(참조용)"로 축소하여 중복 관리 비용 감소

[2] 카테고리별 분석 결과

A) 드래프트
- 상태: 분리 구조가 명확하여 즉시 통합 필요 없음
- 근거: 연도별 클래스(2026/2027/2028+), 시스템(컴바인/드래프트데이), 정의서(능력치)로 역할이 분리됨

B) 내러티브
- 통합 대상 1: 내러티브_부상트레이드.txt
  → CAT-02(부상), CAT-03(트레이드) 전용 원본 파일과 내용 축이 중복
  → 조치: 통합 인덱스 파일로 전환 (원본: 내러티브_부상.txt, 내러티브_트레이드.txt)

- 통합 대상 2: 내러티브_미디어리그국제.txt
  → CAT-11(미디어/소셜), CAT-12(리그/국제) 전용 원본 파일과 내용 축이 중복
  → 조치: 통합 인덱스 파일로 전환 (원본: 내러티브_미디어소셜.txt, 내러티브_리그국제.txt)

- 유지: 내러티브_경기플레이오프.txt, 내러티브_수상레거시.txt
  → 각각 2개 CAT를 의도적으로 묶은 메인 설계 문서이므로 유지

C) 로스터
- 상태: 동부/서부로 기능 분할이 명확하여 통합 불필요

D) 룰
- 상태: 섹션 번호 기준으로 카테고리 분리가 안정적이며 중복 통합 필요 없음

[3] 최종 운영 원칙
- 카테고리 단일 원본 파일에만 실제 룰/이벤트 본문을 추가
- 혼합 파일은 크로스카테고리 탐색용 인덱스로만 사용
- 신규 문서 추가 시 CAT 번호와 기준 원본 파일을 헤더에 명시
